## What was done

1. Defined the project (see `Notes/vision.md`): a fixed benchmark —
   predict P(yea) per member × rollcall — with canonical splits, a fixed
   primary metric (test log loss), a leaderboard, and a roadmap from
   baselines → ideal points → bill-text forecasting → signal extraction →
   autonomous experiment loop.
2. Scaffolded the repo per SOUL.md conventions; copied CLAUDE.md and SOUL.md
   into the repo; git init.
3. `Code/00_download_data.py`: downloaded Voteview members, rollcalls,
   parties, and votes for congresses 101–119 (41 files, 317MB) with a
   manifest (URL, access time, bytes, sha256). All sizes verified vs remote.
4. `Code/01_build_votes.py`: built clean panel — 11,053,847 member×rollcall
   rows (10,602,001 with yea/nay labels), 35,442 rollcalls, 10,430
   member-congress records, 1989-01-03 → 2026-06-09. Zero merge misses,
   zero duplicates. Checks saved to `Modified Data/checks/build_checks.json`.
5. `Code/02_make_splits.py`: canonical hash-based 80/10/10 split
   (regimeA_seed42), content-hashed. All 35,442 rollcalls retain train votes.
6. `Code/harness.py` + `Code/baselines.py` + `Code/run_benchmark.py`:
   evaluation harness (log loss, accuracy, AUC, Brier, APRE, contested
   stratum; structural leakage guard — labels stripped before predict) and
   three baselines.
7. `Code/tests/test_harness.py`: 18 checks (exact metric values, contested
   logic, baseline ordering on synthetic party-line data, leakage guard,
   unseen-key fallbacks, split determinism). All pass.
8. Ran the benchmark (results in README and
   `Modified Data/results/leaderboard.csv`): party_rollcall_rate reaches
   0.206 test log loss / 91.7% accuracy / 0.70 APRE — the bar for Module 2.

## Anomalies investigated

- 111 unique senators in the 111th Congress: real (mid-term replacements —
  Obama, Biden, Clinton, Salazar to the administration, plus deaths).
  Loosened a too-strict sanity bound.
- Per-rollcall yea tallies match official `yea_count` on only ~80% of
  rollcalls; mismatches are median 1 vote, max 6 — known clerk-tally vs
  individual-record discrepancy. Accepted; documented in decisions.md.
