## What was done (P2b: literature baselines — in progress)

1. `Code/models_litbaselines.py`: three literature baselines for the
   horse race Andy requested:
   - `gb_spatial_tfidf` / `gb_spatial_emb2` — Gerrish-Blei 2011 two-stage
     analog: 1D ideal points fit on train; ridge maps text (TF-IDF->SVD
     era-matched, or MiniLM v2 embeddings) to per-rollcall discrimination/
     difficulty; eval rollcalls scored with text-predicted parameters.
     Ridge alpha picked on an internal 10% slice of train rollcalls by
     downstream vote log loss (never val/test).
   - `kraft_bilinear_16d` — Kraft-Jain-Rush 2016 analog via TextTower
     with meta_features off, no mq offset, no MLP, no calibration.
     Documented deviations all favor the baseline (modern embeddings,
     sponsor-alignment term, k=16, temporal early stop).
   - `nominate_context_logit` — strongest no-text null: logistic
     regression on frozen DW-NOMINATE dims x congress x question bucket
     x sponsor alignment (usable on congressout118: unseen-congress
     dummies zero out, NOMINATE main effects carry).
2. `Code/tests/test_litbaselines.py`: 7 synthetic checks, all passing —
   incl. a shuffled-text control (gb gain collapses without real text)
   and Bayes-floor proximity checks. One test threshold initially
   mis-set (world's Bayes floor too close to constant); fixed by
   strengthening the synthetic signal, not by loosening the check.
3. **SIGKILL mystery, root-caused after several wrong theories.** Every
   full-scale GBSpatial run died with exit 137; theories tested and
   discarded in order: (a) system jetsam pressure alone — a real
   JetsamEvent existed but kills were too systematic; (b) per-process
   memory cap — 16GB probe fine; (c) torch-MPS-then-sklearn/BLAS
   ordering — CPU-only run died identically. Live RSS monitoring showed
   the process itself ballooning to 22.8GB; line-level instrumentation
   pinned the death between two adjacent lines in `_vote_log_loss`.
   **Root cause: a silent numpy broadcast bug.** For k=1, sklearn Ridge
   ravels the single-column target, `predict` returns 1-D, so the
   text-predicted discrimination array had shape (n,) instead of (n,1),
   and `(x * A)` broadcast (n_votes,1) x (n_votes,) into an
   (n_votes, n_votes) matrix — 2.5TB at production scale. macOS grants
   the allocation lazily; RSS then inflates as pages materialize until
   jetsam SIGKILLs — hence no Python traceback, ever. The synthetic
   test used k=2 (true multi-output keeps predict 2-D) and so never
   exercised the k=1 path. **Fixes:** `_predict_params` with explicit
   reshape + shape asserts at both call sites (fit's alpha loop and
   predict_proba); dict-of-rows replaced by position-indexed arrays;
   a k=1 production-config test added (8/8 checks pass). Test-design
   lesson recorded: test the production configuration, not just the
   convenient one. (The `device`/`min_epochs` params added to
   IdealPoint along the way are kept: min_epochs guards a REAL
   fragility found in the process — early stopping could fire at epoch
   4 with nothing learned on small data, previously masked by device
   RNG luck; default 0 keeps registered models' behavior identical.)
4. **Second bug the device switch exposed:** IdealPoint had no
   min_epochs guard; bilinear factors start near zero, and on small
   synthetic data (one batch/epoch) patience=3 could fire at epoch 4
   with nothing learned — whether it did depended on device RNG luck
   (MPS init escaped, CPU init didn't; the passing MPS test was silent
   luck). Fix: `min_epochs` param, default 0 so all registered
   ideal_point_* benchmark models behave exactly as before; GBSpatial
   uses min_epochs=15. All suites re-run: 7/7 lit-baseline, ideal-point
   recovery (corr -0.976), 19/19 harness.

4. **P2b results (test log loss, randomrc / forecast):** every
   literature baseline loses to the modern stack by a wide margin, and
   none beats the member x question count baseline (0.494 / 0.459):
   - gb_spatial_tfidf 1.054 / 0.803 (raw; AUC 0.69) — worse than
     CONSTANT on log loss in both regimes despite ~66-72% accuracy
   - gb_spatial_tfidf_tcal 0.685 / 0.566 — temperature recovers
     0.24-0.37 but discrimination remains the binding constraint
   - gb_spatial_emb2(_tcal) 0.944/0.651 (0.662/0.556) — modern
     embeddings help the classical architecture only modestly
   - kraft_bilinear_16d 0.522 / 0.497 (AUC 0.79) — best classical
     architecture, still behind count baselines
   - nominate_context_logit 0.587 / 0.533 (AUC 0.71-0.74)
   - champion emb2_mlp_mq_16d_tcal 0.384 / 0.349 (AUC 0.90)
   Fairness caveat recorded: our GB analog is two-stage ridge, not
   their joint generative inference — raw-variant calibration partly
   reflects that simplification; quote the tcal variant as best-case
   classical and AUC (calibration-invariant) for architecture
   comparisons. Paper takeaways: (a) 2011-era models were never
   competitive with trivial member-history rates under proper scoring
   and honest splits — accuracy masked it (GB 72% vs constant 71% on
   forecast); (b) the modern gain decomposes: representation swap
   inside the classical architecture buys 0.15, architecture + member
   offsets + calibration buy the rest.
